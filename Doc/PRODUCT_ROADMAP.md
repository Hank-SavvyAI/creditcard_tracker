## 階段 2：AI 輔助工具（Week 3-4）

### 開發福利快速輸入工具

```typescript
// apps/backend/src/routes/admin/benefit-parser.ts
router.post('/parse-benefit', async (req, res) => {
  const { rawText, cardId } = req.body

  // 使用 OpenAI API 解析
  const parsed = await openai.chat.completions.create({
    model: "gpt-4",
    messages: [{
      role: "system",
      content: "你是信用卡福利解析專家，將非結構化文字轉換成結構化資料"
    }, {
      role: "user",
      content: `解析以下信用卡福利：\n${rawText}`
    }]
  })

  return res.json({ benefits: parsed })
})
```

### 功能
1. 管理員後台新增「快速解析」功能
2. 貼上銀行官網文字
3. AI 自動產生結構化資料
4. 人工審核修正
5. 一鍵儲存

### 預期效果
- 輸入時間：30min → 5min
- 準確率：95%+
- 可快速擴充到 50+ 張卡

## 法律與合規

### ✅ 合法做法
- 手動輸入公開資訊
- 使用官方 API
- 用戶貢獻內容
- 引用並標註來源

### ❌ 避免風險
- 大規模自動爬蟲
- 繞過反爬蟲機制
- 未經授權的資料抓取
- 侵犯著作權

### 資料來源標註
每筆福利資料都應標註：
```typescript
{
  source: "國泰世華官網",
  sourceUrl: "https://...",
  lastVerified: "2025-11-04",
  verifiedBy: "admin" | "user:123"
}
```

## 風險評估

### 高風險 ⚠️
- 大規模爬蟲 → 法律問題
- 資料不準確 → 用戶信任度

### 中風險 ⚡
- AI 解析錯誤 → 需人工審核
- 眾包品質 → 需審核機制

### 低風險 ✅
- 手動輸入精選卡片
- 使用官方公開資料
- 標註資料來源

---

## 結論

**推薦策略：階段式推進**

1. **現在**：手動輸入 Top 20（快速驗證）
2. **短期**：AI 輔助工具（提升效率）
3. **中期**：眾包機制（擴大規模）
4. **長期**：選擇性自動化（維持更新）

**不推薦**：一開始就做全自動爬蟲
- 法律風險高
- 維護成本高
- 不確定產品市場契合度
